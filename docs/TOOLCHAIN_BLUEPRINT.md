## B. Required MCP Servers

### github (GitHub API)

- **Why:** Agents need to query issues, PRs, releases, GHCR manifests, and push commits/branches during upgrade workflows.
- **Package:** `@modelcontextprotocol/server-github`
- **Install:** `npm install -g @modelcontextprotocol/server-github`
- **Config path:** `.vscode/mcp.json` → `servers.github`
- **Auth:** Requires `GITHUB_TOKEN` (personal access token — see Section J)
- **Recommended allow rules:** repos (`Anthias-Bellforge`), read issues/PRs, read/write branches
- **Recommended deny rules:** delete repos, manage org-level settings

```json
{
  "servers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${env:GITHUB_TOKEN}"
      }
    }
  }
}
```

### filesystem (Local Filesystem)

- **Why:** Agents need to read/write project files, scaffold directories, and inspect build artifacts without shelling out.
- **Package:** `@modelcontextprotocol/server-filesystem`
- **Install:** `npm install -g @modelcontextprotocol/server-filesystem`
- **Config path:** `.vscode/mcp.json` → `servers.filesystem`
- **Recommended allow rules:** workspace root only
- **Recommended deny rules:** `~/.ssh`, `~/.gnupg`, OS system directories

```json
{
  "servers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "c:/Users/ronal/OneDrive - Tulsa Adventist Academy/Documents/GitHub/Anthias-Bellforge"
      ]
    }
  }
}
```

### docker (Docker MCP)

- **Why:** Agents that build, test, and debug Docker containers (image builder, compose test stacks) need programmatic access to the Docker daemon.
- **Package:** `docker/mcp-server` (official Docker MCP)
- **Install:** Via Docker Desktop → MCP settings, or: `docker run -d --name docker-mcp -v /var/run/docker.sock:/var/run/docker.sock ghcr.io/docker/mcp-server`
- **Auth:** Docker daemon socket — no additional token needed if Docker Desktop is installed.

### redis (Redis Inspector MCP)

- **Why:** Agents debugging Celery tasks, WebSocket channel-layer state, and viewer pub/sub need to query Redis keys directly.
- **Package:** `@modelcontextprotocol/server-redis` (community)
- **Install:** `npm install -g mcp-server-redis`
- **Config:**

```json
{
  "servers": {
    "redis": {
      "command": "npx",
      "args": ["-y", "mcp-server-redis"],
      "env": {
        "REDIS_URL": "redis://localhost:6379"
      }
    }
  }
}
```

### playwright (Browser Automation MCP)

- **Why:** Integration tests use Selenium/Splinter, and agents need browser automation for UI validation of the React frontend.
- **Package:** `@playwright/mcp`
- **Install:** `npm install -g @playwright/mcp`

```json
{
  "servers": {
    "playwright": {
      "command": "npx",
      "args": ["-y", "@playwright/mcp"]
    }
  }
}
```
